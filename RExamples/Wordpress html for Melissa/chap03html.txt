<h3>Example 3.1. <a href="../wp-content/data/chapter03/chap03e1GlidingSnakes.csv">Gliding snakes</a></h3>
<em><b>Sample mean</b>, <b>standard deviation</b>, <b>variance</b> and <b>coefficient of variation</b> of undulation rate of 8 gliding paradise tree snakes, in Hz</em>.

Read and inspect the data.
<pre>snakeData &lt;- read.csv(url("http://whitlockschluter.zoology.ubc.ca/wp-content/data/chapter03/chap03e1GlidingSnakes.csv"))
head(snakeData)
</pre>
<strong>Draw a histogram</strong> of the data.
<pre>hist(snakeData$undulationRateHz, right = FALSE)
</pre>
Commands for a fancier histogram are shown [tippy title="here" width = "680" header = "off"]
<pre>hist(snakeData$undulationRateHz, right = FALSE, las = 1, col = "firebrick", 	
	breaks = seq(0.8,2.2,by=0.2), xlab = "Undulation rate (Hz)", 
	ylab = "Frequency", main = "")
</pre>
[/tippy].

Calculate the <strong>mean, standard deviation</strong> and <strong>variance</strong> of the undulation rates.
<pre>mean(snakeData$undulationRate)
sd(snakeData$undulationRate)
var(snakeData$undulationRate)
</pre>
Calculate the <strong>coefficient of variation</strong>.
<pre>100 * sd(snakeData$undulationRate)/mean(snakeData$undulationRate)
</pre>

<h3>Table 3.1-2. <a href="../wp-content/data/chapter03/chap03t1_2ConvictionsFreq.csv">Numbers of convictions</a></h3>
<i><b>Mean and standard deviation from a frequency table</b>. The data are from Chapter 2</i>.

Read and inspect the data. It is a frequency table of the number of convictions.
<pre>convictionsFreq &lt;- read.csv(url("http://whitlockschluter.zoology.ubc.ca/wp-content/data/chapter03/chap03t1_2ConvictionsFreq.csv"))
head(convictionsFreq)
</pre>
Calculate the <b>mean and standard deviation from the frequency table</b>. First, use the <code>rep</code> command to "repeat" each value in the table, the number of times according to its frequency. Here, we store the result in <code>convictions</code>.
<pre>convictions &lt;- rep(convictionsFreq$convictions, convictionsFreq$frequency)
</pre>
Then, calculate mean and standard deviation on the result.
<pre>mean(convictions)
sd(convictions)
</pre>

<h3>Example 3.2. <a href="../wp-content/data/chapter03/chap03e2SpiderAmputation.csv">Spider running speed</a></h3>
<em><b>Median</b>, <b>interquartile range</b>, and <b>box plot</b> of running speed (cm/s) of male <i>Tidarren</i> spiders. We also include below the <b>cumulative frequency distribution</b> of running speed before amputation.
</em>

Read and inspect the data. The data are in "long" format. One variable indicates running speed, and a second variable gives treatment (before vs after amputation). Therefore, every individual spider is on two rows, once for its before-amputation measurement and one for its after-amputation measurement.
<pre>spiderData &lt;- read.csv(url("http://whitlockschluter.zoology.ubc.ca/wp-content/data/chapter03/chap03e2SpiderAmputation.csv"))
head(spiderData)
</pre>
<strong>Box plot</strong> of the data. Begin by ordering the treatment levels so that the "before" amputation measurements come before the "after" measurements in the plot.
<pre>spiderData$treatment &lt;- factor(spiderData$treatment, levels = c("before", "after"))
boxplot(speed ~ treatment, data = spiderData)
</pre>
Instructions for a fancier box plot, made with additional options, is shown [tippy title="here" width = "680" header = "off"]
<pre>par(bty = "l")
boxplot(speed ~ treatment, data = spiderData, ylim = c(0,max(spiderData$speed)),
	col = "goldenrod1", boxwex = 0.5, whisklty = 1, las = 1,
	xlab = "Amputation treatment", ylab = "Running speed (cm/s)")
</pre>
[/tippy].

<strong>Extract the before-amputation data</strong> using <code>subset</code>. Note the double equal sign "==" needed in the logical statement, which indicates "is equal to" in R. Save the result in a new data frame named <code>speedBefore</code>.
<pre>speedBefore &lt;- subset(spiderData, treatment == "before") 
speedBefore
</pre>
Calculate the <strong>sample median</strong> of before-amputation running speed.
<pre>median(speedBefore$speed)
</pre>
<strong>Calculate the first and third quartiles</strong> of before-amputation running speed (0.25 and 0.75 quantiles). Type 5 reproduces the method we use in the book to calculate quartiles.
<pre>quantile(speedBefore$speed, probs = c(0.25, 0.75), type = 5)
</pre>
Determine the <strong>interquartile range</strong> of before-amputation running speed. Type 5 reproduces the method we use in the book to calculate quartiles.
<pre>IQR(speedBefore$speed, type = 5)
</pre>
<b>Draw a cumulative frequency distribution</b> of running speed before amputation (Figure 3.4-1).
<pre>plot( ecdf(speedBefore$speed), verticals = TRUE,
	ylab = "Cumulative relative frequency", 
	xlab = "Running speed before amputation (cm/s)")
</pre>
Instructions for a slightly more polished cumulative frequency distribution are [tippy title="here" width = "680" header = "off"]
<pre>par(bty = "l")
plot( ecdf(speedBefore$speed), verticals = TRUE,  
	las = 1, main = "", do.points = FALSE,
	ylab = "Cumulative relative frequency", 
	xlab = "Running speed before amputation (cm/s)" )
</pre>
[/tippy].

<h3>Example 3.3. <a href="../wp-content/data/chapter03/chap03e3SticklebackPlates.csv">Stickleback lateral plates</a></h3>
<em>Draw <b>multiple histograms</b> and <b>produce a table of descriptive statistics by group</b> for plate numbers of three stickleback genotypes. We also include a <b>table of frequencies and proportions</b> of stickleback genotypes.</em>

Download and inspect the data.
<pre>sticklebackData &lt;- read.csv(url("http://whitlockschluter.zoology.ubc.ca/wp-content/data/chapter03/chap03e3SticklebackPlates.csv"))
head(sticklebackData)
</pre>
<strong>Draw multiple histograms</strong> of number of plates, one histogram per genotype. Begin by setting the preferred order of the three genotypes in the figure. Here, we use the <code>lattice</code> package to draw the histograms, so it must be loaded first.
<pre>sticklebackData$genotype &lt;- factor(sticklebackData$genotype, 
	levels = c("MM","Mm","mm"))
library(lattice)
histogram(~ plates | genotype, data = sticklebackData, breaks = seq(0,70,by=2), 
	  layout = c(1, 3), col = "firebrick")
</pre>
Commands to draw multiple histograms in base R, without using the <code>lattice</code> package, are [tippy title="here" width = "680" header = "off"]
<pre>oldpar = par(no.readonly = TRUE) # make backup of default graph settings
par(mfrow = c(3,1), las = 1, oma = c(4, 6, 2, 6), mar = c(2, 5, 4, 2)) # adjust margins
hist(sticklebackData$plates[sticklebackData$genotype == "MM"], right = FALSE, 
	  breaks = seq(0,70,by=2), main = "MM", col = "firebrick",
	  las = 1, ylab = "Frequency")
hist(sticklebackData$plates[sticklebackData$genotype == "Mm"], right = FALSE, 
	  breaks = seq(0,70,by=2), main = "MM", col = "firebrick",
	  las = 1, ylab = "Frequency")
hist(sticklebackData$plates[sticklebackData$genotype == "mm"], right = FALSE, 
	  breaks = seq(0,70,by=2), main = "MM", col = "firebrick",
	  las = 1, ylab = "Frequency")
mtext("Number of lateral plates", side = 1, outer = TRUE, padj = 1.5)
par(oldpar) # revert to default graph settings
</pre>
[/tippy]. This method is more tedious, but allows for easier addition of options.

<strong>Make a table of descriptive statistics by group</strong> using <code>tapply</code>. The commands below assume that the variables contain no missing (NA) elements. We need to calculate all the statistics, one at a time. Save them so that we can put them all together in a table afterward.

To begin, get the sample sizes by group (genotype):
<pre>n &lt;- tapply(sticklebackData$plates, INDEX = sticklebackData$genotype, FUN = length)
n
</pre>
Next, calculate the mean number of plates by group. Let's round the results to 1 decimal place to make it easier to read them when we place into a table. To accomplish this, set <code>digits = 1</code> in the <code>round</code> function.
<pre>meanPlates &lt;- tapply(sticklebackData$plates, INDEX = sticklebackData$genotype, 
	FUN = mean)
meanPlates &lt;- round(meanPlates, digits = 1)
meanPlates
</pre>
Repeat for medians.
<pre>medianPlates &lt;- tapply(sticklebackData$plates, INDEX = sticklebackData$genotype, 
	FUN = median)
medianPlates
</pre>
Repeat for standard deviations, including rounding.
<pre>sdPlates &lt;- tapply(sticklebackData$plates, INDEX = sticklebackData$genotype, FUN = sd)
sdPlates &lt;- round(sdPlates, 1) 
sdPlates
</pre>
Finally, the interquartile range by group.
<pre>iqrPlates &lt;- tapply(sticklebackData$plates, INDEX = sticklebackData$genotype, 
	FUN = IQR, type = 5)
iqrPlates
</pre>
<strong>Make the table by assembling all the results in a new data frame</strong> (data frames can be used as tables for display purposes).
<pre>sticklebackTable &lt;- data.frame(genotype = names(n), n = n, 
	mean = meanPlates, median = medianPlates, 
	sd = sdPlates, iqrange = iqrPlates)
sticklebackTable
</pre>
We can <b>make a table of frequencies and proportions</b> of the stickleback genotypes (Table 3.5-1). Below, we first generate a frequency table of genotypes (the <code>dnn</code> argument names the variable in the table).
<pre>sticklebackFreq &lt;- table(sticklebackData$genotype, dnn = "genotype")
sticklebackFreq
</pre>
We then convert the table to a data frame so that we can include the proportions in the table too.
<pre>sticklebackFreq &lt;- data.frame(sticklebackFreq)
sticklebackFreq
</pre>
Finally, we <strong>calculate the proportions</strong> and put them into the data frame. To convert frequencies to proportions, divide the frequencies by the sum of the frequencies.
<pre>sticklebackFreq$proportion &lt;- sticklebackFreq$Freq / sum(sticklebackFreq$Freq)
sticklebackFreq
</pre>
The table would look even nicer if you round the proportions before including them in the table (give this a try).
