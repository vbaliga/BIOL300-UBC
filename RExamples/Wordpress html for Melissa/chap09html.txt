<h3>Example 9.2. <a href="../wp-content/data/chapter09/chap09e2AspirinCancer.csv">Aspirin and cancer</a></h3>
<em><b>Odds ratio</b> and <b>relative risk</b> to estimate association between aspirin treatment and cancer incidence in a 2x2 contingency table</em>.

Read and inspect the data.
<pre>cancer &lt;- read.csv(url("http://whitlockschluter.zoology.ubc.ca/wp-content/data/chapter09/chap09e2AspirinCancer.csv"))
head(cancer)
</pre>
<strong>2 x 2 contingency table</strong> showing association between aspirin treatment and cancer incidence (Table 9.2-1).
<pre>cancerTable &lt;- table(cancer$cancer, cancer$aspirinTreatment)
cancerTable
</pre>
<strong>Mosaic plot</strong> showing the association (Figure 9.2-1).
<pre>mosaicplot( t(cancerTable), col = c("firebrick", "goldenrod1"), cex.axis = 1, 
	sub = "Aspirin treatment", ylab = "Relative frequency", main = "")
</pre>
<strong>Calculate odds ratio</strong> using the <code>epitools</code> package. Install the package if you haven't already done so (this needs to be done just once per computer, after you've installed R). To use, load the <code>epitools</code> package (this needs to be done just once per R session).
<pre>install.packages("epitools", dependencies = TRUE)
library(epitools)
</pre>
The formulas for the odds ratio and relative risk in the book assume that the contingency table has a layout exactly like <code>cancerTable</code>, namely
<pre>#            treatment control
#  sick          a        b
#  healthy       c        d
</pre>
This layout works for the <code>oddsratio</code> function.
<pre>oddsratio(cancerTable, method = "wald")
</pre>
The resulting output of <code>oddsratio</code> includes a lot of incidental computations. To get clean output only for the odds ratio, including the 95% confidence interval, use the following instead.
<pre>oddsratio(cancerTable, method = "wald")$measure[-1,]
</pre>
For step-by-step calculations of the odds ratio and confidence interval, see [tippy title="here" width = "680" header = "off"]
<pre># First assign the labels a,b,c, and d to the table, as on page 240.
# Then plug the values into the formula for odds ratio
x = as.vector(cancerTable)
names(x) = c("a","c","b","d")
orHat = unname(x["a"] * x["d"]) / (x["b"] * x["c"])
unname(orHat)

# Standard error of log odds ratio
seLnOR = sqrt(1/x["a"] + 1/x["b"]+ 1/x["c"] + 1/x["d"])
unname(seLnOR)

# 95% confidence interval
Z = qnorm(1 - 0.05/2) # when rounded, is 1.96
ciLnOR = c(log(orHat) - Z * seLnOR, log(orHat) + Z * seLnOR)
ciOR = exp(ciLnOR) # transform back to ratio scale
names(ciOR) = c("lower","upper")
ciOR
</pre>
[/tippy].

<strong>Calculate relative risk</strong> using the <code>epitools</code> package. The layout expected by the <code>riskratio</code> function is the complete opposite of the layout used in the book. To use the command with a contingency table in book style (such as <code>cancerTable</code>), we need to flip (transpose) the table and reverse the order of the rows. We can do this all at once with the following arguments to the <code>riskratio</code> function.
<pre>library(epitools)
riskratio(t(cancerTable), rev = "both", method = "wald")
</pre>
Notice that the result differs slightly from the relative risk value given in the book for these same data (1.007) because rounding error here is reduced.

For clean output only for the relative risk, including the 95% confidence interval, use the following command:
<pre>riskratio(t(cancerTable), method = "wald", rev = "both")$measure[-1,]
</pre>
For step-by-step calculations of the relative risk, see [tippy title="here" width = "680" header = "off"]
<pre>x = as.vector(cancerTable)
names(x) = c("a","c","b","d")
phat1 = x["a"]/(x["a"] + x["c"])
phat2 = x["b"]/(x["b"] + x["d"])
rrHat = unname(phat1/phat2)
unname(rrHat)
 
# Standard error
seLnRR = sqrt( 1/x["a"] + 1/x["b"] - 1/(x["a"]+x["c"]) - 1/(x["b"]+x["d"]) )
unname(seLnRR)

# 95% confidence interval
Z = qnorm(1 - 0.05/2) # when rounded, is 1.96
ciLnRR = c(log(rrHat) - Z * seLnRR, log(rrHat) + Z * seLnRR)
ciRR = exp(ciLnRR) # transform back to ratio scale
names(ciRR) = c("lower","upper")
ciRR
</pre>
[/tippy].

<h3>Example 9.3. <a href="../wp-content/data/chapter09/chap09e3ToxoplasmaAndAccidents.csv">Toxoplasma and driving accidence</a></h3>
<em><b>Odds ratio</b> to estimate association between Toxoplasma infection and driving accidence in a 2 x 2 <strong>case-control study</strong></em>.

Read and inspect the data.
<pre>toxoplasma &lt;- read.csv(url("http://whitlockschluter.zoology.ubc.ca/wp-content/data/chapter09/chap09e3ToxoplasmaAndAccidents.csv"))
head(toxoplasma)
</pre>
<strong>2 x 2 contingency table</strong> (Table 9.3-1).

toxTable
</pre>

oddsratio(toxTable, method = "wald")
oddsratio(toxTable, method = "wald")$measure[-1,] # clean output

<h3>Example 9.4. <a href="../wp-content/data/chapter09/chap09e3ToxoplasmaAndAccidents.csv">Worm gets bird</a></h3>
<em><b>χ<sup>2</sup> contingency test</b> to test association between trematode infection status of killifish and their fate (eaten or not eaten) in the presence of predatory birds</em>.

Read and inspect the data.
<pre>worm &lt;- read.csv(url("http://www.zoology.ubc.ca/~schluter/WhitlockSchluter/wp-content/data/chapter09/chap09e4WormGetsBird.csv"))
head(worm)
</pre>
Set the preferred order of infection categories in tables and graphs.
<pre>worm$infection &lt;- factor(worm$infection, 
	levels = c("uninfected", "lightly", "highly"))
</pre>
<strong>Contingency table</strong> (Table 9.4-1 ). The <code>addmargins</code> function adds the row and column sums to the display of the table.
<pre>wormTable &lt;- table(worm$fate, worm$infection)
addmargins(wormTable)
</pre>
<strong>Mosaic plot</strong> (Figure 9.4-1).
<pre>mosaicplot( t(wormTable), col = c("firebrick", "goldenrod1"), cex.axis = 1, 
	sub = "Infection status", ylab = "Relative frequency")
</pre>
<strong>χ<sup>2</sup> contingency test</strong>. We include the argument <code>correct = FALSE</code> to avoid Yates' correction. This has no effect except in 2 x 2 tables, but we keep it here for demonstration purposes.
<pre>saveChiTest &lt;- chisq.test(worm$fate, worm$infection, correct = FALSE)
saveChiTest
</pre>
The <strong>expected frequencies under null hypothesis</strong> are included with the results, but aren't normally shown. Type <code>saveChiTest$expected</code> to extract them. Include the <code>addmargins</code> function if you want to see the row and column sums too.
<pre>addmargins(saveChiTest$expected)
</pre>
<b><i>G</i>-test</b> applied to the worm-gets-bird data (Section 9.6). R has no simple, built-in function to carry out the <i>G</i>-test with goodness-of-fit data. Code for a command <code>g.test</code> by Brent Larget is available <a href="http://www.stat.wisc.edu/~st571-1/gtest.R">here</a>. Below, we source this code, which then allows you to use his function <code>g.test</code>.
<pre>source("http://www.stat.wisc.edu/~st571-1/gtest.R")
g.test(wormTable)
</pre>

<h3>Example 9.5. <a href="../wp-content/data/chapter09/chap09e5VampireBites.csv">Vampire bat feeding</a></h3>
<b>Fisher's exact test</b> of association between estrous status of cows and whether cows were bitten by vampire bats.

Read and inspect the data.
<pre>vampire &lt;- read.csv(url("http://www.zoology.ubc.ca/~schluter/WhitlockSchluter/wp-content/data/chapter09/chap09e5VampireBites.csv"))
</pre>
<strong>Contingency table</strong> (Table 9.5-1).
<pre>vampireTable &lt;- table(vampire$bitten, vampire$estrous)
vampireTable
</pre>
<strong>Expected frequencies under null hypothesis of independence</strong>. R complains because of the violation of assumptions. Just in case you hadn't noticed.
<pre>saveTest &lt;- chisq.test(vampire$bitten, vampire$estrous, correct = FALSE) 
saveTest$expected
</pre>
Fisher's exact test. The output includes an estimate of the odds ratio, but it uses a different method than is used in our book.
<pre>fisher.test(vampire$bitten, vampire$estrous)
</pre>
